# 前端最终打磨方案

状态：待确认草案  
项目：The Fifth Stone  
审查日期：2026-05-15

## 1. 目标

这是进入后端开发前的最后一轮前端收尾。

这次不建议重做整个网站。现在的品牌方向已经成立：神话灵感、珠宝饰品、柔和仪式感、以商品转化为核心。最后一轮前端修改应该让页面更像一个准备上线的品牌官网和商店，而不是继续扩展新功能。

本轮目标：

- 保留现有品牌气质。
- 去掉明显的模板感和 AI 生成感。
- 补齐真实电商前端应有的交互状态。
- 修正移动端、购物车、表单、导航等最后体验问题。
- 完成后暂停前端，开始后端。

## 2. 当前优点

- 品牌氛围清楚：象牙白、柔金色、神话图像、产品摄影、衬线标题整体是统一的。
- 核心页面已经齐全：首页、商品/系列页、故事页、联系页、FAQ、政策页。
- 产品图片资产比较完整：五种颜色都有主图、佩戴图、细节图、包装图。
- `npm run build` 已通过，说明当前代码基础是稳定的。
- 基础购物车流程可用：添加商品、数量调整、抽屉购物车、总价、移动端购物车。
- SEO 基础已经做好：标题、描述、canonical、结构化数据、sitemap、法律链接。

## 3. 主要问题

### A. 整体过于米金色，层次略弱

现在网站很柔和，但很多区块都在使用相近的 ivory / cream / gold 视觉语言。优点是统一，缺点是页面往下滚动后容易变得平、淡、缺少商品重点。

建议方向：

- 保留象牙白作为主背景。
- 加强深色文字、石头颜色、区块分隔和图片对比。
- 不要把网站做成五颜六色。五种石头颜色应该只在产品相关区域出现，而不是到处散落。

### B. 字体系统需要最后升级

当前字体是 Cormorant Garamond + Inter。Cormorant Garamond 很适合神话和珠宝气质，但 Inter 会让正文和按钮显得比较通用。

建议方向：

- 保留 Cormorant Garamond 用于标题、故事和仪式感文案。
- 将 Inter 替换为更有性格的无衬线字体，例如 Outfit，或接近 Satoshi 气质的字体栈。
- 收紧长段落宽度和行高，让阅读更高级。
- 减少小号全大写标签的滥用，尤其是类似 `01`、`02` 这种装饰编号。

可能涉及文件：

- `index.html`
- `src/style.css`

### C. 部分 UI 细节仍然有生成感

审查中发现一些容易显得模板化的元素：

- 概念区和包装内容里使用 `✦`。
- 按钮、状态和故事区里有文本箭头、文本勾号。
- 评论区用 `★` 和 `☆` 字符。
- 多处等分卡片网格：三个概念卡、五个商品卡、三个评论卡、五个石头描述卡。
- 信任区使用 `01`、`02`、`03`、`04` 这类编号。

建议方向：

- 把文本符号替换为小型内联 SVG 或 CSS 图形。
- 评论区不要直接使用星号字符，可以改成更克制的评分文字、短线、或自定义 CSS 星形。
- 五个商品卡必须保留，因为产品概念就是五块石头，但布局可以更有设计感，比如桌面端做 2 + 3 的节奏，移动端保持清晰单列或双列。
- 概念区不要继续做三个完全一样的卡片，可以改成更像品牌宣言或编辑版式的模块。

可能涉及文件：

- `src/pages/home.ts`
- `src/pages/product.ts`
- `src/pages/story.ts`
- `src/style.css`

### D. 首页后半部分节奏需要打磨

首页首屏很强，品牌和商品图片都有第一眼信号。但往下滚动后，部分区块逐渐变得模板化。

建议修改：

- 保留现有 hero 方向和首屏图片。
- 优化故事预告区，避免滚动后固定导航栏压住视觉重心。
- 将三个概念卡改为更高级的品牌理念区。
- 将商品预览区从五个等宽卡片改成更有节奏的商品展示。
- 评论区减少普通三卡片排布的感觉。
- 邮件订阅区增加真实表单校验、成功状态和错误状态。

优先级：

- 高：概念区、商品预览区、订阅表单状态。
- 中：评论区。
- 低：少量文案微调。

### E. 商品页需要更像正式电商页面

商品页图片强，购物车也能用，但购买路径还可以更正式。

建议修改：

- 保留当前商品图库和颜色选择。
- 让第一屏更紧凑，方便同时看到图片、标题、价格、颜色、加入购物车、信任信息和卖点。
- 优化移动端底部 sticky bar，避免它和页面里的价格重复得太明显。
- 加入购物车后使用正式的行内状态，不再用符号式的 `Added` 状态。
- 目前后端还没做，所以结账按钮不要用 `alert()`，改成品牌化的“结账即将开放”或“后端接入中”状态。
- 减少商品页中部过大的空白。
- 石头描述、材质、评论等区域减少一排排卡片的重复感。

可能涉及文件：

- `src/pages/product.ts`
- `src/main.ts`
- `src/cart.ts`
- `src/style.css`

### F. 故事页氛围好，但太淡

故事页的神话氛围是对的，但首屏和几个故事屏幕太浅，文字存在感稍弱。移动端故事页也有较大留白，CTA 出现得偏低。

建议修改：

- 保留电影感和神话背景图。
- 略微增强文字对比度。
- 把 `100vh` 改为更适合移动浏览器的 `100dvh` 或 `100svh`。
- 压缩移动端首屏空白，让故事页看起来更有意图。
- 如果改动范围可控，把 `story.ts` 里的大段页面内联样式迁移到 `style.css`，让页面代码更好维护。

可能涉及文件：

- `src/pages/story.ts`
- `src/style.css`

### G. 导航和移动菜单需要封版处理

Header 已经可用，但还缺少正式站点的最后细节。

建议修改：

- 当前页面导航项增加 active 状态，并设置 `aria-current`。
- 移动菜单打开/关闭动画更清楚。
- 点击任何导航项后都应稳定关闭移动菜单。
- 增加隐藏的 skip-to-content 链接，改善键盘用户体验。
- 扩展 `focus-visible` 样式，覆盖所有主要交互控件。

可能涉及文件：

- `src/components/Header.ts`
- `src/router.ts`
- `src/style.css`

### H. Loading、Error、Empty、Form 状态需要统一

现在部分状态仍然依赖内联样式或浏览器默认行为。

发现的问题：

- Router loading 和 error 用的是内联 HTML 样式。
- 购物车空状态有内联样式。
- Checkout 使用 `alert()`。
- Newsletter 只检查是否包含 `@`，然后直接改按钮文字。
- Contact 表单主要依赖浏览器原生校验。

建议修改：

- 建立统一的 loading 和 error 样式。
- 购物车空状态增加更完整的品牌化文案和返回商品页入口。
- 用行内提示替代 `alert()`。
- Newsletter 增加 `aria-live` 状态文本。
- Contact 表单成功/错误状态与品牌视觉统一。

可能涉及文件：

- `src/router.ts`
- `src/main.ts`
- `src/pages/home.ts`
- `src/components/PolicyPageLayout.ts`
- `src/style.css`

## 4. 最后一轮前端修改范围

### 后端开始前必须做

1. 字体和设计 token 清理。
2. 去掉主要可见 UI 里的 raw text 装饰符号。
3. 优化移动菜单和当前页面导航状态。
4. 替换 `alert()`、内联 loading、empty、error 状态。
5. 商品页购买流程打磨。
6. 首页卡片和网格节奏打磨。
7. 故事页对比度和视口单位清理。
8. 桌面端和移动端浏览器验证。

### 时间允许可以做

1. 让评论区更高级。
2. 增加一个统一、轻量的纹理或 grain 质感。
3. 用克制的 CSS 动效改善区块进入节奏。
4. 清理确认无用的 Vite starter 资产。

### 本轮不要做

1. 不做支付后端。
2. 不接真实 checkout。
3. 不改法律政策文案，除非明确要求。
4. 不整体替换产品图片。
5. 不迁移框架。
6. 不引入重型动画库。

## 5. 建议执行顺序

1. 设计 token 和字体
   - 更新字体 import 和 CSS 变量。
   - 调整正文对比度、行高、标签样式。

2. 全局交互状态
   - 按钮、链接、focus ring、active nav、移动菜单。
   - Loading、error、empty 状态。

3. 首页打磨
   - 概念区。
   - 商品预览区。
   - 评论区和 newsletter 状态。

4. 商品页打磨
   - Hero 区信息密度。
   - 加入购物车状态。
   - Checkout 未开放状态。
   - 移动端 sticky bar。
   - 下方区块卡片节奏。

5. 故事页打磨
   - 对比度、间距、`100dvh` 清理。
   - 视情况迁移 `story.ts` 内联样式。

6. 验证
   - `npm run build`
   - 桌面端检查：首页、商品页、故事页、联系页。
   - 移动端检查：首页、商品页、故事页、移动菜单、购物车。
   - 核心交互检查：导航、加入购物车、数量调整、关闭购物车、订阅表单、联系表单。

## 6. 验收标准

- 构建通过。
- 已检查页面没有明显浏览器 console error。
- 购物流程里不再出现 `alert()`。
- 主要可见 UI 不再使用 raw text 装饰符号。
- Header 能显示当前页面状态。
- 移动菜单打开和关闭清晰稳定。
- 购物车空状态和有商品状态都完整。
- 商品页移动端 sticky bar 不遮挡关键内容。
- 主要页面保持 The Fifth Stone 的统一品牌气质。
- 前端可以阶段性冻结，进入后端开发。

## 7. 建议结论

建议批准一次聚焦的最终前端打磨。

最稳妥的做法是只执行“后端开始前必须做”的范围。这样能明显提升页面完成度，同时避免在切后端前进行高风险大改。
